    Overview

    Introduction

    Generic test automation framework for acceptance testing and acceptance test-driven development (ATDD)

    • Enables easy-to-use tabular syntax for creating test cases in a uniform way
    • Allows using keyword-driven, data-driven and behavior-driven (BDD) approaches
    • Provides easy-to-read reports and logs in HTML format

    ● The core framework is Implemented using Python

    • Runs also on Jython (JVM) and IronPython (.NET)
    • Can be extended by test libraries implemented either with Python or Java
    • Can create new higher-level keywords from existing ones using the same syntax that is used for creating test cases
    • Provides support for Selenium for web testing.
    • Other languages supported via a remote interface

    ● Open source

    • Hosted on GitHub, released under Apache License 2.0
    • Developed at Nokia Networks and it is nowadays sponsored by Robot Framework Foundation
    • Rich ecosystem and very active community

    More detail about Robot framework: refer to http://robotframework.org/robotframework/latest/RobotFrameworkUserGuide.html

     

    Test libraries

    Test libraries provide the actual testing capabilities to Robot Framework by providing keywords. There are several standard libraries that are bundled in with the framework, and galore of separately developed external libraries that can be installed based on your needs. Creating your own test libraries is a breeze.

    Source: http://robotframework.org/#test-libraries

    Demos with Python

    2 demos from http://robotframework.org/#documentation

    http://robotframework.org/#documentation had few demos. In this session, I will present 2 demos (Web demo and Robot demo in Python). You just follow the guides on 2 links below and my notes on MAC OS

    • Web Demo: https://bitbucket.org/robotframework/webdemo
    • Robot Demo: https://bitbucket.org/robotframework/robotdemo

    Before running demos according to the guide, make sure things in "Preconditions" sections are correct: installed python, pip (The main benefit of using pip is that it automatically installs all dependencies needed by the library. Other nice features are easy upgrading and support for UN-installation), robot framework and selenium2library on Mac OS.

    1. Install Python: By default, in MAC OS have python with version 2.7 (you can check with command "python" on Terminal). Or you can download and install new version at https://www.python.org/downloads/
    2. Install pip (The main benefit of using pip is that it automatically installs all dependencies needed by the library. Other nice features are easy upgrading and support for UN-installation): 
      1. Download get-pip.py from https://pip.pypa.io/en/stable/installing/
      2. Open terminal: cd Downloads and type "sudo python get-pip.py"
        1. You need to type your password (with command "sudo") before downloading and installing
        2. Finally, pip will be installed successfully:
          1. Installing collected packages: pip, wheel
          2. Successfully installed pip-8.1.2 wheel-0.29.0
        3. To check whether pip installed on mac or note: type "pip" will see the list of command on pip
    3. Install robot: via pip, you can install robot framework and selenium2library easily with command:
      1. sudo pip install robotframework
      2. sudo pip install robotframework-selenium2library

    Then you can run the demo following the guide on links above. Below are my notes:

    WebDemo

    • cd Downloads/WebDemo -> cd to directory contains demoapp/server.py
    • python demoapp/server.py -> run server file to start port 7272 -> Demo server starting on port 7272.
    • robot Downloads/WebDemo/login_tests -> run all tests insides
    • robot login_tests/valid_login.robot robot --test InvalidUserName --loglevel DEBUG login_tests

    RobotDemo

    • cd Downloads/RobotDemo
    • robot keyword_driven.robot
    • robot --name Robot --loglevel DEBUG keyword_driven.robot data_driven.robot gherkin.robot -> To execute all test case files in a directory recursively

    Finally, open report.html file to view report on each demo

    Notes on 2 demos

    • The keyword-driven approach works well for normal test automation (CalculatorLibrary.py, data_driven.robot, valid_login.robot), but gherkin style might be even better if also business people need to understand tests (gherkin_login.robot, gherkin.robot)
    • If the same workflow needs to repeated multiple times, it is best to use to the data-driven approach (ex: web demo - test case invalid-login.robot).
    • After viewing report.html, know the keyword on each step belong to which test library (for example, test case "Calculation error" on data-driven.robot file: CalculatorLibary.Should Cause Error; BuildIn.Should Be Equal). Then know how to check keyword document
    • All keyword documents are available on http://robotframework.org/#test-libraries (such as BuildIn, Selenium2Library for Java - keyword document, etc...)
    • What information should manage in resource file (ex: resource.robot - from web demo): server, browesr, delay time, valid data, url of pages, etc... Then use this resource on test cases ("resource" at Setting - valid-login.robot file from web demo)
    • Use "Document" to describe test suite 
    • run robot file (test cases) must begin with "robot" command

    Another tutorial with Python: https://www.youtube.com/channel/UCgcU5MOQ7Q-huwBdGYIcYfQ

     

     

     

     

    Getting start with Robot Framework and Java (a simple project that includes the robot framework maven dependencies)

    Create JavaRobot project built in Maven

    • The Robot Framework With Java - Getting Started - YouTube
    • Getting started with the RobotFramework and Java - Automation Testing Made Easy
    1. Create maven project in Eclipse
    2. Edit pom.xml file
    • Add the plugin to your build and  you can override the dependency robot-framework

    Source: http://robotframework.org/MavenPlugin/

    <project>
      ..
      <plugin>
        <groupId>org.robotframework</groupId>
        <artifactId>robotframework-maven-plugin</artifactId>
        <version>1.4.6</version>
        <executions>
          <execution>
            <goals>
               <goal>run</goal>
            </goals>
          </execution>
        </executions>
        <dependencies>
          <dependency>
            <groupId>org.robotframework</groupId>
            <artifactId>robotframework</artifactId>
            <version>2.8.7</version>
          </dependency>
        </dependencies>
      </plugin>
        ..
</project>
    • Add dependency Selenium2Library for JAVA from Test Libraries

    Source: https://github.com/MarkusBernhardt/robotframework-selenium2library-java#readme

    <dependency>
    <groupId>com.github.markusbernhardt</groupId>
    <artifactId>robotframework-selenium2library-java</artifactId>
    <version>1.4.0.8</version>
    <scope>test</scope>
</dependency>
